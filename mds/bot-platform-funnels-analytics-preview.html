Воронки и аналитика — что нужно достроить

Движок и события уже работают. Не хватает удобной обёртки над ними — визуального конструктора и нормальных дашбордов. Ниже — целевой дизайн с моками.

Сейчас
🟡 Минимум
  • ✅ Движок: state-machine, 10 типов нод, шаблоны переменных
  • ✅ 3 шаблона флоу (приветствие, лид-магнит, квиз)
  • ✅ JSON-редактор с валидацией Zod
  • ⚠️ Нет визуального drag-n-drop — собирать руками в JSON
  • ✅ События пишутся в tg_events (20 типов)
  • ✅ Базовая статистика: подписчики, сообщения, топ-теги
  • ⚠️ Нет дашбордов конверсии по нодам
  • ⚠️ Нет UTM-разрезов (firstTouch есть в БД, но нет отчёта)
  • ⚠️ Нет когорт retention
Целевая картина
🟢 Полноценно
  • 🎨 Визуальный canvas: drag-n-drop ноды, стрелки между ними
  • 🎨 Палитра нод слева, properties справа
  • 🎨 Inline-предпросмотр сообщений и кнопок
  • 🎨 Подсказки про {{vars.x}} на лету
  • 📊 Дашборд конверсии: % дроп-офф между нодами
  • 📊 UTM-pivot: выручка/подписки в любом разрезе
  • 📊 Когорты: % retention по неделям подписки
  • 📊 Per-flow-аналитика: время в каждой ноде, дроп
  • 📊 Per-broadcast: доставлено/прочитано/клики/отписки

План работ — что построить

Готовое ядро (движок + событийный стрим) переживёт оба апгрейда. Это слой UI и слой агрегаций — без изменений API/БД (только добавочные таблицы для агрегатов).

Конструктор воронок
Визуальный canvas на React Flow
  • 📦 Добавить пакет @xyflow/react
  • 🎨 10 кастомных React-Flow-нод с цветовой кодировкой
  • 🧩 Палитра слева, properties справа (как в макете выше)
  • 🔄 Сохраняем тот же JSON-формат (tg_flows.graph) — engine ничего не знает про UI
  • ↶ Undo/redo через локальную history-стек
  • 🧪 Тестовый запуск без БД — симулятор шагов в браузере
  • 📍 Подсветка ноды, в которой сейчас находятся активные runs
  • ⚙️ Drag-handles на каждой ноде, авто-layout (dagre) для импорта шаблонов
Оценка: 1.5–2 дня на одного фуллстек. Зависимости: @xyflow/react, dagre.
Аналитика
5 дашбордов + материализация агрегатов
  • 📊 Recharts уже в зависимостях — графики готовы
  • 📦 Добавить таблицу tg_event_daily — агрегаты по дню/типу/боту
  • ⏰ Cron-тик уже есть — досыпать пересчёт агрегатов каждые 5 мин
  • 🎯 Дашборд воронки: SQL по flow.node_executed группой по nodeId
  • 🎯 Дашборд UTM: firstTouchSlug/lastTouchSlug уже в БД
  • 🎯 Когорты: groupBy неделя подписки × неделя message.received
  • 🎯 Per-broadcast метрики — уже есть в tg_broadcasts, нужны графики
  • 💰 Подключить выручку (если есть Payment-нода) — TBD по платежам
Оценка: 2–2.5 дня. Зависимости: уже все есть (recharts, prisma raw).
Что построим в каком порядке?
  1. Аналитика-MVP (4–6 ч): дашборды Обзор + UTM + Per-flow конверсия по tg_events. Без когорт.
  2. Визуальный редактор (6–8 ч): React Flow, 10 типов нод, properties-panel, undo/redo. JSON формат не меняется.
  3. Когорты + Per-broadcast аналитика (3–4 ч).
  4. Агрегатор (2 ч): таблица tg_event_daily + cron-пересчёт.

💬 Скажите «делай», и я начну с пункта 1 — конверсии воронки. Это самая полезная штука, чтобы видеть «где утекают деньги» прямо в админке.
